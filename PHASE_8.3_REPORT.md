## Controller Logic

### Data Loading Process
1. **Extract Month**: Extract YYYY-MM from attendance_date (YYYY-MM-DD)
2. **Build Query**: Start with User query with eager loading (department, designation, shift)
3. **Apply Department Filter**: If department_id provided, add where clause
4. **Apply Shift Filter**: If shift_id provided, add where clause
5. **Load Employees**: Execute query to get filtered employees
6. **Load Attendance Data**: Load all AttendanceMonth records in one query using `whereIn`
7. **Build Attendance Array**: Loop through employees, extract day data from attendance_json
8. **Apply Status Filter**: If status provided, filter out non-matching records
9. **Pass to View**: Return data to blade

### Query Optimization
- **Eager Loading**: `User::with(['department', 'designation', 'shift'])` loads related models in 1 query
- **Bulk Attendance Load**: `AttendanceMonth::whereIn('user_id', $employees->pluck('id'))` loads all attendance in 1 query
- **Key By**: `->keyBy('user_id')` enables O(1) lookup instead of O(n)
- **Total Queries**: 3 queries (Users + AttendanceMonth + SiteInfo) regardless of employee count
- **Avoids N+1**: No queries inside loops

### Validation
- Attendance Date required (HTML5 validation)
- Department optional
- Shift optional
- Status optional
- No SQL errors (uses Eloquent ORM with null coalescing)
- No undefined index errors (uses null coalescing operator `??`)
- No undefined offset errors (uses array access with null coalescing)
- No null property access errors (checks for null before accessing)
